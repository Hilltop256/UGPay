## Architecture Decisions

1. **React Router over TanStack Router**: Simpler setup, sufficient for current needs
2. **Zustand over Redux**: Less boilerplate, built-in TypeScript support, perfect fit for auth state
3. **Sonner for toasts**: Lightweight, accessible, shadcn-compatible
4. **Vitest over Jest**: Native TypeScript/ESM support, faster execution
5. **Lazy loading**: All pages use `React.lazy()` for code splitting
6. **Protected routes via component wrapper**: Inline guard in `AuthenticatedLayout` rather than separate component
7. **eslint-disable for Router.tsx**: Router files naturally export non-component values (the router config object)
